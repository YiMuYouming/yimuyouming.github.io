# 弈沐资本 Portal 2.0

## 定位

Portal 2.0 是弈沐资本的对外展示窗口，不是纯内部工作台。

核心表达：

- 用 AI 提高研究密度
- 用复盘压实判断证据
- 由人负责最终取舍
- 用收益曲线和交易记录验证结果

## 首页结构

1. 首屏品牌区
2. 今日市场状态
3. 收益记录
4. AI 复盘闭环
5. 人机协同交易框架
6. 研究与规则沉淀
7. 常用入口

## 模块说明

### 首屏品牌区

说明“AI 增强的 A 股短线趋势交易体系”，保留三个核心入口：

- 查看收益记录
- 阅读最新复盘
- 了解协同框架

### 今日市场状态

独立放在收益记录前，避免挤占首屏呼吸感。

同步字段：

- 上证
- 深证
- 创业
- 成交额
- 涨跌比
- 涨跌停
- 情绪值

### 收益记录

保留日、周、月切换和指数对照能力。默认展示账户曲线、TWR、相对指数、最大回撤、仓位等核心指标。

### AI 复盘闭环

首页展示复盘统计和近期 6 篇复盘卡片。每张卡片展示：

- 日期
- 复盘类型
- 标题
- 上证
- 涨跌比
- 涨跌停
- 情绪值

### 人机协同交易框架

首页展示框架摘要和流程图。详细页承载完整方法论、边界说明和与传统量化、经验短线、龙虎榜跟随的区别。

### 研究与规则沉淀

分为三类：

- 交易认知
- 交易规则体系
- 投研报告

首页展示摘要，详情页承载完整卡片和报告内容。

### 常用入口

集中保留内部入口：

- 最新复盘
- 全部复盘
- 收益曲线
- 交易体系
- 投研资源

## 自动同步契约

### `tools/sync_pnl_data.py`

职责：

- 从 Hermes 云端 bridge 拉取 PnL 数据
- 从 Hermes 云端 bridge 拉取市场快照
- 更新 `index.html` 内的收益曲线数据
- 更新 `MARKET_SNAPSHOT_START/END` 市场状态模块

默认命令：

```bash
python3 tools/sync_pnl_data.py
```

本地调试：

```bash
python3 tools/sync_pnl_data.py --source local
```

### `tools/convert_review.py`

职责：

- 把 Vault 复盘 Markdown 转为 `review-notes/YYYY-MM-DD.html`
- 更新首页最新复盘入口
- 更新 AI 复盘闭环统计
- 更新近期 6 篇复盘卡片
- 更新可审计交易链路日期和记录跨度
- 更新 `review-notes/index.html`

命令：

```bash
python3 tools/convert_review.py <vault_md_path>
```

同一天重复同步时必须刷新内容，但不能重复增加统计数量。
